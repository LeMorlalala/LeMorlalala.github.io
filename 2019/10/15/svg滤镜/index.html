SVG滤镜

2019-10-15   svg 

svg 滤镜相关知识

一些背景

  1. 滤镜称得上是 SVG 最强大的功能了，它允许你给图形(图形元素和容器元素)添加各种专业软件中才有的滤镜特效。这样你就很容易在客户端生成和修改图像了。而且滤镜并没有破坏原有文档的结构，所以维护性也很好。

  2. 滤镜用 filter 元素定义；需要使用的时候，在需要滤镜效果的图形或容器上添加 filter 属性，引用相关滤镜即可。

  3. 起始和结束filter标记之间就是执行我们想要的操作的滤镜基元。每个基元有一个或多个输入，但是只有一个输出。一个输入可以是原始图形（SourceGraphic）、图形的阿尔法通道（黑色部分，即不透明的部分，被指定为 SourceAlpha），或者是前一个滤镜基元的输出。当只对图形的形状感兴趣而不管颜色的时候，阿尔法源是有用的，它会避免阿尔法和颜色相互作用。

  4. 使用滤镜的目的：滤镜应该用来帮助和增强我们的消息，而不是反过来让消息更不明显，明智地用一两个滤镜会有好处，而太多的滤镜几乎总是会弱化消息。

一些定义

<feFlood>提供了一个纯色区域用于组合或者合并。它们很像<feOffset>，允许我们在一系列滤镜基元内执行某些常见的操作，而不是在主图形中创建额外的 svg 元素。我们只需提供flood-color和flood-opacity然后滤镜会完成其他工作。

<feMorphology>元素允许我们对图形进行”瘦身”或者加厚，可以指定operator值为erode来给图形进行瘦身，或者指定为dilate来加厚图形。radius属性用来告诉我们增厚或者变薄了多少。

<feMerge>基元包裹一个<feMergeNode>元素列表，其中每个元素都指定一个输入。这些输入按照它们出现的顺序一个堆叠在另一个上面。

<feGaussianBlur>表示生成投影，用 stdDeviation 属性指定它的模糊度，这个数值越大，模糊度越大。如果提供两个由空格分隔的数字，第一个表示 x 方向的模糊度，第二个作为 y 方向的模糊度。

<feComposite>元素接受两个输入源，假设我们已经给前一个滤镜基元指定了 result=”A”和 result=”B”，下面是主要用的 in out over 的解释，还有 xor atop arithmetic:

  • <feComposite operator="over" in="A" in2="B" /> 生成的 A 层叠在 B 层上面，正如 feMergeNode 做的那样，事实上，feMergeNode 仅仅是指定 over 操作的一种便利的快捷方式。

  • <feComposite operator="in" in="A" in2="B" /> A 的一部分重叠在 B 的不透明区域，类似于蒙版效果，但是这个蒙版仅仅基于 B 的阿尔法通道，而不是它的颜色亮度。

  • <feComposite operator="out" in="A" in2="B" /> 结果是 A 的一部分位于 B 的不透明区域外部

用一朵发来测试这些滤镜基元

feGaussianBlur

feGaussianBlur 基元的输入源我们使用 SourceAlpha 时，则表示我们只是模糊黑色部分，若不使用 feMerge 基元叠加上原始图形，出来的只是模糊的阿尔法通道图形。我们也可以用 SourceGraphic 作为输入源，出现的效果就是模糊的花朵源图形。

feOffset

feOffset 基元接受 feGaussianBlur 返回的结果 in=”blur”可写可不写，它的偏移位置由 dx 和 dy 的值指定，然后将结果位图存储在 offsetBlur 名字下面。

feColorMatrix

投影在花朵上的效果很好，但应用给文本时则完全不理想，看例子在 jsbin 上的演示；但如果文本周围有一个发光区域，看起来会更好看

feColorMatrix 允许我们修改任意像素点的颜色或者阿尔法值，当他的 type 是 matrix，value 必须设置为 4x5 的矩阵。每行代表一个代数方程，定义了如何计算输出的 R G B A 值，每行数字分别乘以输入像素的 R G B A 和常量值，然后加在一起得到最后的输出值。上图中设置的变换是将所有不透明的区域绘制成相同的颜色，此时可以忽略颜色和常量，只要设置阿尔法的值即可。

对于复杂的 feColorMatrix 值的设置，公式是这样的，它能够对每个颜色通道进行编辑，类似 photoshop 中的通道编辑

feImage

至此，我们都使用原始图形或者阿尔法通道作为滤镜的输入源。SVG 的<feImage>元素允许我们使用任意的 jpg、png、svg 文件，或者带有 id 属性的 SVG 元素作为滤镜输入源，还是以之前那朵发作为例子：

feComposite

over 和 feMerge 的作用类似，in 表示源 A 重叠在源 B 的不透明区域，out 则营造了一种”镂空”的效果

feDisplacementMap

这个滤镜使用第二个输入的颜色值决定在第一个输入中移动像素的距离，我们用 xChannelSelector 指定应该用哪个颜色通道来影响像素的 x 坐标，用 yChannelSelector 属性指定用哪个颜色通道来影响 y 坐标。这些选择器的合法值是 R、G、B、A。还必须指定移动像素的距离；scale 属性用于指定缩放因子。如果不指定这个属性，滤镜什么都不做。

feTurbulence

feTurbulence允许我们通过使用 Ken Perlin 开发的方程，生成大理石，云彩等人工纹理效果，要指定以下属性。

  • type turbulence 和 fractalNoise 之一，后者显示更平滑

  • baseFrequency 属性值的数字越大，结果颜色的变化越快，数值必须大于 0 小于 1，提供两个数字的情况则第一个表示 x 的频率，第二个作为 y 的频率。

  • numOctaves 这是噪音函数使用的数值 生成最终结果时应该加上这个数值，数值越大，纹理粒度越细，默认值为 1

  • seed 这个滤镜使用的随机生成器的种子，默认值为 0 改变它可以得到一些不同的效果

给 path 加上内部发光的效果

内边框只是把原图叠加一层需要设置的内边框颜色，在此基础上，再叠加一层变瘦的图形，颜色为原图形的默认颜色。

内发光的效果使用了 feComposite 的 operatpr 属性，in 表示覆盖在不透明区域，out 表示覆盖在不透明区域外。首先我们定义一个颜色层，并使用 out 的特性填充在原图的不透明区域，就相当在整个 svg 区域内把原图挖空，那么当前是一个外部填充了#000，但原图被抠掉的区域（为透明）。到第 ② 步位置，则是给图加上了高斯模糊，并且叠了一层绿色上去。高斯模糊针对于图形的边且根据 stdDeviation 数值的大小形成模糊的效果，我们这里的模糊是想用来做内部的发光效果。第 ③ 步也是最重要的一步，把刚刚设置的外部区域重叠在原始图形的不透明区域，则相当于 ② 步骤出来的图形剪成原始图形的样子，值保留了边缘的高斯模糊和内部的透明。此时使用 feMergeNode 在原图的颜色叠上 ③ 出来的元素，则看上去为内发光的效果。
