定义比例尺

x 轴使用分段比例尺，定义域可以是一组数据，x 轴根据数组的长度自动进行均分，其输出值为对应数组中的值，值域是左侧间距到 svg 的宽度

y 轴使用线性比例尺，其中定义域的值可以表示为 x，值域的值可以表示为 y，其中的函数关系是 y=mx+b,定义域通常是 0 到传入数据的最大值，这边最大值*1.08 是希望 y 轴最高值 40 的展示在最高点偏下一些，则扩大了定义域。

有一点需要注意的是，y 轴的值域和定义域是相反的。这是由于 y 轴的正方向是由上往下，当前比例尺的计算规则是，假设定义域是 x（图标的值），值域是 y，当 x 是 0 的时候，y 是当前画图区域的最大高度;当 x 是最大值时，y 是 0 。那么，假设在 x 轴最高点画一条水平线，调用 yScale 算出来的值其实可以理解为从水平线往下的距离，柱状图柱子的高度则可以设置为画图区域减去 yScale。

创建一个饼图生成器

pie 根据指定的 data 数组生成一组对象数组，其中每个对象包含每个传入的数据经过计算后的角度信息,包括起始角度，结束角度等。 也包含额外的 argements,比如上述的 category 信息。对象数组的长度和传入的 data 数组长度一致，并且一一对应。

value指定当前弧对应的值

startAngle endAngle表示绘制整个饼图的起始角度和结束角度

sortValues根据上面设置的 value 字段，取值进行比较，不改变 index 及画图的顺序，只是改变弧的起始角度

创建一个弧生成器

弧生成器会根据数据的角度、半径等信息，返回一个 path 字符串。

innerRadius outerRadius表示饼图的内径和外径，若饼图大小需要平铺整个 svg,则可以取宽高之中较小的除 2 作为外半径

arc.centroid(argements)表示弧的中间点，方便于对饼图的 labels 进行定位。中间点大致被定义为(startAngle + endAngle)/2 和 (innerRadius + outerRadius) / 2，可以直接用于 translate 后方

绘制饼图

d3.schemeCategory10实质上是 d3 提供的一个颜色数组，包含十个分类颜色的数组，其他还有 d3.schemeDark2,d3.schemePaired, d3.schemeSet3 等参见官方 api,一般来说都是自己定义颜色数组

classed第一个参数传要设置的 class 名称，第二个参数可以是布尔型的 true 或者 false,也可以传入 fn,判断之后返回 true 或者 false

添加 path 标签，d 属性的值可以传入弧生成器或者直线生成器等 fn,自动生成 path 数据，也可以自行通过M L中间加点绘制

transition表示为指定的选择集增加一个过渡,即使用动画，后面可以设置 duration(100),表示过渡到指定状态的时间，以毫秒为单位;delay(100),表示过渡的延时时间（毫

attrTween(name[,factory])表示使用自定义插值器在给定的属性之间过渡，官网解释是：如果 factory 不为 null,factory 会为每个选中的元素进行调用，并依次传递当前元素绑定的数据 d 以及索引 i, 函数内部 this 指向当前 DOM 元素。返回的插值器会在过渡过程中的每一帧进行调用，并依次传入 eased(缓动) 时间 t, 通常情况下在 [0, 1] 范围内。最后插值器返回的值将会被用来设置为当前属性值。例如：a 是 0, b 是 20, 当 t=0.5 的时候，值为 10
